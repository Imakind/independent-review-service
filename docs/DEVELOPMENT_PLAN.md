# План разработки

## Паттерн этапа

Каждый этап описывается одинаково:

1. Цель — зачем делаем.
2. Результат — что должно появиться.
3. Работы — что именно реализовать.
4. Критерий готовности — как проверить завершение.

## 1. Концепция поиска

### Цель

Определить, как система ищет сайты, приложения, аккаунты и внутренние объекты.

### Результат

Единая логика поиска:

```text
platform_key + identifier_type + normalized_value
```

### Работы

- Разделить объекты на сайт, приложение, аккаунт, сервис внутри сайта, профиль продавца.
- Ввести родительские и дочерние объекты.
- Описать типы идентификаторов: `domain`, `url`, `username`, `phone`, `internal_id`.
- Описать типы совпадений: `exact_match`, `domain_match`, `platform_match`, `similar_match`, `ambiguous_match`.

### Критерий готовности

Система может объяснить, что найдено: сайт, приложение, аккаунт или объект внутри платформы.

## 2. Модель данных

### Цель

Создать базовую структуру хранения.

### Результат

Схема для объектов, идентификаторов, отзывов и жалоб.

### Работы

- Создать `objects`.
- Создать `object_identifiers`.
- Создать `reviews`.
- Создать `reports`.
- Добавить связь объекта с родителем.

### Критерий готовности

Можно сохранить сайт, приложение, аккаунт внутри приложения и отзыв к нему.

## 3. Нормализация идентификаторов

### Цель

Приводить разные формы одного объекта к единому виду.

### Результат

Модуль нормализации.

### Работы

- Нормализовать телефоны.
- Нормализовать домены.
- Нормализовать URL.
- Нормализовать username.
- Нормализовать ссылки `t.me`, `instagram.com`, `wa.me`.
- Убирать UTM и tracking-параметры.

### Критерий готовности

Одинаковые объекты в разных форматах дают один `normalized_value`.

## 4. Backend API

### Цель

Сделать центральную бизнес-логику.

### Результат

API для поиска, создания отзывов и жалоб.

### Работы

- Реализовать поиск.
- Реализовать создание объекта.
- Реализовать создание отзыва.
- Реализовать жалобу.
- Реализовать карточку объекта.
- Добавить валидацию.

### Критерий готовности

Через API можно найти объект, создать отзыв и получить карточку результата.

## 5. Telegram-бот

### Цель

Сделать первый пользовательский интерфейс.

### Результат

Рабочий Telegram-бот для MVP.

### Работы

- Реализовать `/start`, `/help`, `/rules`.
- Реализовать `/check`.
- Реализовать `/review`.
- Сделать пошаговые сценарии с кнопками.
- Показывать карточку объекта.
- Добавить жалобу на отзыв.

### Критерий готовности

Пользователь может через Telegram проверить объект и оставить отзыв.

## 6. Поиск

### Цель

Сделать понятную и безопасную выдачу.

### Результат

Поиск с типами совпадений и уровнем уверенности.

### Работы

- Сначала искать точное совпадение.
- Потом искать родительский сайт или приложение.
- Потом искать похожие объекты.
- Для username без платформы возвращать неоднозначные результаты.
- Не смешивать одинаковые username из разных приложений.

### Критерий готовности

Пользователь видит не просто результат, а степень совпадения.

## 7. Отзывы

### Цель

Реализовать основную пользовательскую ценность.

### Результат

Механизм создания и просмотра отзывов.

### Работы

- Добавить тип отзыва: `positive`, `neutral`, `negative`.
- Добавить категорию.
- Добавить текст.
- Добавить доказательства.
- Добавить статусы: `pending`, `published`, `rejected`, `hidden`.

### Критерий готовности

Отзывы сохраняются, отображаются и связаны с конкретным объектом.

## 8. Модерация

### Цель

Снизить риск клеветы, спама и публикации персональных данных.

### Результат

Минимальная система модерации.

### Работы

- Добавить статус `pending`.
- Добавить жалобы.
- Добавить причины жалоб.
- Добавить ручное скрытие.
- Добавить rate limit.
- Помечать отзывы без доказательств.

### Критерий готовности

Администратор может скрыть, отклонить или оставить спорный отзыв.

## 9. Админ-панель

### Цель

Дать администратору управление спорными данными.

### Результат

Минимальная панель управления.

### Работы

- Список новых отзывов.
- Список жалоб.
- Просмотр карточки объекта.

### Результат

Набор обязательных правил и политик.

### Работы

- Правила публикации отзывов.
