# Концепция поиска

## Цель

Поиск должен отвечать на три вопроса:

- есть ли отзывы именно об этом объекте;
- является ли найденный объект точным совпадением или похожим;
- есть ли связанные отзывы о родительском сайте или приложении.

## Основная формула

```text
platform_key + identifier_type + normalized_value
```

## Типы объектов

- `website` — сайт или домен.
- `app` — приложение или платформа.
- `website_profile` — объект внутри сайта.
- `app_profile` — аккаунт внутри приложения.
- `phone` — телефон как самостоятельный идентификатор.
- `service` — сервис без устойчивой платформенной структуры.
- `person_alias` — псевдоним или имя, только как слабый сигнал.

## Типы идентификаторов

- `domain`;
- `url`;
- `url_path`;
- `username`;
- `phone`;
- `deep_link`;
- `internal_id`;
- `service_name`.

## Типы совпадений

### exact_match

Найден тот же нормализованный идентификатор внутри той же платформы.

### domain_match

Точного объекта внутри сайта нет, но есть отзывы о домене.

### platform_match

Точного аккаунта внутри приложения нет, но есть отзывы о приложении.

### similar_match

Найдены похожие объекты. Это не доказательство совпадения.

### ambiguous_match

Ввод неоднозначен. Например, username без платформы.

## Сайт и объект внутри сайта

Пример:

```text
https://example.com/seller/ivan123?utm_source=tg
```

Нормализация:

```text
platform_key: website
identifier_type: url_path
normalized_value: example.com/seller/ivan123
parent: example.com
```

Если точного совпадения нет, поиск должен проверить родительский домен `example.com`.

## Приложение и аккаунт внутри приложения

Пример:

```text
t.me/seller123
```

Нормализация:

```text
platform_key: telegram
identifier_type: username
normalized_value: seller123
```

Пример:

```text
@seller123
```

Если платформа неизвестна:

```text
platform_key: unknown
identifier_type: username
normalized_value: seller123
match_type: ambiguous_match
```

Такой результат нельзя считать точным.

## Приоритет идентификаторов

1. Внутренний неизменяемый ID.
2. Публичная ссылка.
3. Username внутри известной платформы.
4. Телефон.
5. Название аккаунта или сервиса.

## UX-правило Telegram-бота

Бот обязан показывать пользователю тип совпадения:

- точное совпадение;
- совпадение только по сайту;
- совпадение только по приложению;
- похожий результат;
- неоднозначный username.

Пользователь не должен принимать похожий объект за доказанно тот же объект.
